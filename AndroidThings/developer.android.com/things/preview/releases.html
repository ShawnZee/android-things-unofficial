    欢迎加入 IoT 开发者 Google+ 社区 向其他Android Things开发者提问并进行讨论。

    开发者预览2

    日期： 2017年2月
    Build 编号： NIG40
    Play 服务 : 10.0.0

    预览 APIs Javadoc 说明 。

    这篇预览发布指导 开发者和早期适配器 在支持的硬件平台上进行开发和兼容性测试所使用。 请注意以下关于本预览的整体指南

    • 本次发布可能会对所支持硬件有各种 稳定性问题 欢迎报告你所发现的问题。
    • 在本预览中并不是所有APIs都可用。一些已知问题所涉及的API可能无法使用，它们被记录在了文档中。
    • 开发者预览2 支持Intel Edison, Intel Joule, NXP Pico和Raspberry Pi 3开发板

    Preview 2中的新增

    支持Intel Joule

    Android Things 现在支持 Intel® Joule compute module. 获取更多关于此设备和 开发套件 功能

    本地 peripheral API

    现在可通过使用 Android NDK中 Native PIO library 实现C/C++代码对的peripheral I/O支持 你可以从 示例页获取新的Native PIO的使用示例。

    USB 音频支持

    无板载模拟音频功能的设备现在支持USB麦克风和音响，来实现音频的录制和回放。对于 Preview 2，现在支持以下平台

    • Intel® Edison
    • Intel® Joule
    • Raspberry Pi

    TensorFlow 示例

    我们创建了一些示例，演示如何在Android Things设备上使用TensorFlow。这些示例演示了获取相机，物体识别，图像分类以及语音合成。

    请访问 示例页来获取更多信息.

    Peripheral 管理报告

    开发者现在可以监控设备的peripheral的活动端口状态来进行开发期间的调试，使用 dumpsys 指令: 

    $ adb shell dumpsys com.google.android.things.pio.IPeripheralManager

    已知问题

    • 系统能源管理现在不可用。设备将不会被挂起，同时唤醒锁机也是不必要的。
    • 蓝牙API无法使用。
    • USB API目前无法使用。
    • 危险的权限 需要设备下次启动才会生效。他们包括新app安装和在已有app中新的 <用户级权限>的元素。
    • Google Play 服务在第一次启动时需要2-3分钟的预优化。此时，App的安装会被阻止，直到这一过程完成。
    • 硬件图形加速(OpenGL)目前无法使用。API依赖的功能(如 WebView ) 目前无法使用。

    Peripheral I/O

    • Peripherals 在使用 close() 之后并不会清除或重置。 输出将保持他们的状态，同时串口将继续传输之前缓存的数据。
    • GPIO pins 若在上次重启之后被edge 触发器启用并当作输入，那么便不能被当做输出。

    User Drivers

    • 用户传感器目前不能被手动注册。他们将在app进程结束时被自动注销。
    • 用户传感器只支持 持续检测并变化的 传感器。One-shot和特殊报告模式可能会表现的不符预期。

    Edison

    • RESET: Arduino breakout上的RESET按钮将临时使你的板子进入一个不一致的状态，此时GPIO pin叫做GPxx，而不是IOXX，直到断电这个状态才会结束。 所以，可以通过插拔电源来进行重启，而不是使用RESET按钮。

    Joule

    • 相机 相机目前还不支持。
    • I/O: Edge出发器目前只支持以下的GPIO端口： DISPLAY_0_BIAS_EN, DISPLAY_0_BKLT_EN, DISPLAY_0_RST_N, FLASH_RST_N, FLASH_TORCH, FLASH_TRIGGER.
    • I/O: GPIO初始化配置的共享pin口需要在下次重启后才能被用作任何其他功能（SPI，UART等）

    Pico

    • 网络： 以太网卡目前不可用
    • I/O: 当pin口被设置为输出时Gpio.getValue() 将总是返回 false

    Raspberry Pi

    • 网络： 若以太网卡连接了一个不能上网的网络，Wi-Fi将不能连接intel网
    • 相机： 一个全新的若没有超过一个的目标输出装置 CameraCaptureSession 将不能被创建。
    • 相机 在CameraCaptureSession中的第一次请求 总是等待两张图片。这可以引起每一次CaptureRequest 在一次会话中从前一次的捕捉中返回缓存的frame。
    • I/O: 共享pin口 BCM13/PWM1 和 BCM18/PWM0 若在上次重启之后被启用了PWM，那么便不可被用作GPIO。
    • I/O: GPIO pin口 BCM4, BCM5, 和 BCM6 当被用作输入时，电压将被提升到3.3V。
    • 音频: 板载模拟音频将不能与PWM同时使用。

    Developer Preview 1

    日期: December 2016
    Build编号: NIF73/NIF74
    Play Services: 10.0.0

    预览 APIs Javadoc 说明.

    这篇预览发布指导 开发者和早期适配器 在支持的硬件平台上进行开发和兼容性测试所使用。 请注意以下关于本预览的整体指南

    • 本次发布可能会对所支持硬件有各种 稳定性问题 欢迎报告你所发现的问题。
    • 在本预览中并不是所有APIs都可用。一些已知问题所涉及的API可能无法使用，它们被记录在了文档中。
    • Developer Preview 1 支持 the Intel Edison, NXP Pico, 和 Raspberry Pi 3 开发板.

    已知问题

    • 系统能源管理现在不可用。设备将不会被挂起，同时唤醒锁机也是不必要的。
    • 蓝牙 APIs 现在不可用.
    • USB APIs 现在不可用.
    • 危险权限 需要设备下次启动才会生效。他们包括新app安装和在已有app中新的 <用户级权限>的元素。
    • Google Play 服务在第一次启动时需要2-3分钟的预优化。此时，App的安装会被阻止，直到这一过程完成。
    • 当多个动作包含了IOT_LAUNCHER类别的过滤器 系统将显示app选择器，但是在没用显示支持的设备上则无法获取到。Android Things只支持单一的launcher app， 同时这个行为将在未来的release中被关闭。

    Peripheral I/O

    • Peripherals 在使用 close() 之后并不会清除或重置。 输出将保持他们的状态，同时串口将继续传输之前缓存的数据。
    • GPIO pins 若在上次重启之后被edge触发器启用并当作输入，那么便不能被当做输出。

    User 驱动

    • 用户传感器目前不能被手动注册。他们将在app进程结束时被自动注销。
    • 用户的传感器目前只支持 持续检测并变化的 传感器。One-shot和特殊报告模式可能会表现的不符预期。

    Edison

    • 音频: 目前不支持录音.
    • I/O: GPIO pin GP77 被列在 PeripheralManagerService中, 但是无法连接app。
    • RESET: Arduino breakout上的RESET按钮将临时使你的板子进入一个不一致的状态，此时GPIO pin叫做GPxx，而不是IOXX，直到断电这个状态才会结束。 所以，可以通过插拔电源来进行重启，而不是使用RESET按钮。

    Pico

    • 网络: 以太网卡目前不可用.
    • I/O: 当pin口被设置为输出时Gpio.getValue() 总是返回 false

    Raspberry Pi

    • 网络 Wi-Fi 若以太网卡连接了一个不能上网的网络，Wi-Fi将不能连接intel网。
    • 图形: 硬件图形加速目前不可用。
    • 相机: 一个全新的若没有超过一个的目标输出装置 CameraCaptureSession 将不能被创建。
    • 相机: 在CameraCaptureSession中的第一次请求 总是等待两张图片。这可以引起每一次CaptureRequest 在一次会话中从前一次的捕捉中返回缓存的frame。
    • I/O: 共享 pin口 BCM13/PWM1 和 BCM18/PWM0 若在上次重启之后被启用了PWM，那么便不可被用作GPIO。
    • I/O: GPIO pin口 BCM23 和 BCM24 现在将同时被用于控制 BCM23 (physical pin J8-16).
    • I/O: GPIO pin口 BCM4, BCM5, 和 BCM6 当被用作输入时，电压将被提升到3.3V。
